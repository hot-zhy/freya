AI 代码智能与 AI 编程

Coding Agent / Code Intelligence（逐题 Q/A）

Q1：代码模型和通用聊天模型差异？

A1：代码模型更关注语法正确性、仓库上下文一致性和可执行通过率。评估也更偏工程指标，如 pass@k、编译通过率和单测通过率。

Q2：怎么设计“读仓库-改代码-提 PR”的 Agent？

A2：先检索相关代码，再规划改动点，生成补丁后执行测试，失败则自动回退并二次修复。必须加沙箱执行和变更审查，防止越权改动。

Q3：如何降低 AI 改代码引入回归？

A3：设置三层闸门：静态检查、自动化测试、变更范围限制。高风险目录加人工审核和灰度发布，保证可控上线。

Q4：为什么要语义检索 + 结构检索混合？

A4：语义检索擅长找“相似逻辑”，结构检索擅长找“准确符号依赖”。两者结合召回更稳，尤其适合大型代码库。

Q5：代码 Agent 如何评估？

A5：离线看任务基准，线上看真实研发效率，如修复时长、评审轮次、一次通过率。评估要按任务类型分桶，不能只看平均值。

Q6：主要安全风险与治理？

A6：风险包括提示词注入、敏感信息泄露、恶意依赖。治理要做最小权限、密钥扫描、依赖签名、审计追踪和隔离执行。
