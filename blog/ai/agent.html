AI Agent 理论与框架

ReAct / Planning / Tool Use（逐题 Q/A）

Q1：Agent 与 ChatBot 的本质区别？

A1：Agent 具备目标驱动、多步规划、工具调用和反馈纠错能力；普通 ChatBot 多是单轮回答，不具备可执行闭环。

Q2：ReAct 为什么有效？

A2：ReAct 让模型在“推理-行动-观察”中循环，通过工具结果不断修正思路，显著减少闭门推理导致的错误。

Q3：什么时候用 Plan-and-Execute？

A3：任务复杂、步骤长、依赖多时优先使用。先规划再执行能控制 token 成本，并让失败可定位、可回滚。

Q4：Agent 记忆系统怎么设计？

A4：短期记忆放会话上下文，长期记忆放结构化画像与事件。要有写入门槛和过期机制，避免噪声污染决策。

Q5：多 Agent 协作常用哪种模式？

A5：常见是协调者+专家代理。协调者拆解任务并汇总结果，专家处理垂直子任务，同时设置轮次上限防止无穷对话。

Q6：Agent 可靠性怎么做？

A6：关键是权限边界、参数校验、超时重试、补偿机制和审计日志。高风险操作引入人工确认，保证可追责。
