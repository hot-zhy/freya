RAG 进阶实战与 Agentic RAG

流程优化、范式设计、评估落地（面试可复述版）

一、RAG 全链路优化（8 步）

1) 数据接入与清洗（Ingestion）

  • 目标：把原始文档变成可检索、可追踪、可增量更新的知识资产。
  • 优化：去模板噪声、保留标题层级与来源字段、建立版本号和更新时间。
  • 坑点：只保留纯文本，后续权限过滤、时间过滤、归因追溯都会变差。

2) 分块（Chunking）

  • 目标：每个块可召回且语义完整。
  • 优化：先按语义边界切（标题/段落/函数），再控 token，保留 10%~20% overlap。
  • 实践：FAQ、代码、规章文档分开策略，避免“一刀切”。

3) Embedding 与索引

  • 目标：平衡召回率、时延和成本。
  • 优化：同域 embedding + ANN 参数调优（HNSW/IVF）+ 冷热分层索引。
  • 坑点：换 embedding 模型却不重建索引，向量空间会漂移。

4) Query Understanding（查询理解）

  • 目标：把自然语言问题转成检索友好表达。
  • 优化：query rewrite、实体补全、同义扩展、多路并行检索。
  • 加分：复杂问题先拆子问题，提高多跳问题命中率。

5) 召回 + 重排（Retrieve + Rerank）

  • 目标：先高覆盖，再高精度。
  • 优化：混合检索（BM25 + Vector）并行召回，cross-encoder 重排到 topK。
  • 工程：先 metadata 过滤再重排，减少算力开销并提升稳定性。

6) 上下文构建（Context Assembly）

  • 目标：给模型“高价值低噪声”证据包。
  • 优化：去重、合并相邻块、按可信度排序、控制 token budget。
  • 建议：每条证据绑定 source id，输出可追溯引用。

7) 生成约束与拒答策略

  • 目标：正确、可解释、可拒答。
  • 优化：提示词约束“仅基于证据回答”；证据不足触发拒答或补检索。
  • 坑点：不设拒答边界会导致高置信幻觉。

8) 反馈闭环（Feedback Loop）

  • 目标：上线后持续提升而不是停留初版。
  • 优化：沉淀失败样本（没召回/召回错/生成错），做回放与回归评测。
  • 策略：优先改检索和重排，再考虑模型侧改造。

二、Agentic RAG 常见范式

范式 A：ReAct RAG

Thought → Retrieve → Observe → Refine → Answer，多轮迭代修正；适合多跳问题。要设置轮次上限与超时，防止循环。

范式 B：Plan-and-Execute RAG

先拆子问题，再逐个检索与汇总；适合长任务和报告生成。关键是统一子任务输出 schema，避免汇总失真。

范式 C：Router RAG

先路由再检索，按问题类型分发到 FAQ/代码库/工单库等；多知识域场景效果好。需要 fallback 通道防路由错误。

范式 D：Self-RAG / CRAG

模型先答再自检证据充分性，不足则补检索重答；准确率导向场景常用。阈值过严会拉高时延与成本。

范式 E：Multi-Agent RAG

Planner、Retriever、Verifier、Writer 分工协作；复杂任务更稳。要明确角色边界、共享状态和审计日志。

范式 F：GraphRAG

关系图检索 + 文本证据融合；适合依赖链和因果链问题。优势是关系可解释，代价是图谱维护成本高。

三、RAG 评估怎么做（必须分层）

检索层指标（Retriever）

  • Recall@k：正确证据是否进入 topK（最关键）。
  • MRR/NDCG：正确证据排序质量。
  • HitRate、P95/P99 检索时延、查询类型覆盖度。

生成层指标（Generator）

  • Correctness：答案是否正确。
  • Faithfulness/Groundedness：答案是否被证据支持。
  • Citation Accuracy：引用是否准确。
  • Completeness：回答是否完整。

端到端指标（E2E）

  • 任务成功率、一次解决率、追问率、人工接管率。
  • 成本指标：单问 token 成本、重排成本、平均响应时延。
  • 用户侧：满意度、复用率、留存。

评估落地方法

离线：构建 benchmark（问题-金证据-标准答案）做回归；线上：A/B 实验；回放：失败会话重放验证改进。LLM-as-Judge 可用，但必须配人工抽检校准。

四、面试可直接背诵

30 秒版：“我做 RAG 会按数据、检索、生成三层优化。检索看 Recall@k/MRR，生成看 groundedness/引用准确率，线上看时延和成功率。复杂问题会用 Agentic RAG 迭代检索纠错。”

2 分钟版：“我把链路拆成 8 步：数据清洗、chunk、混合召回、重排、上下文构建、生成约束、反馈回放。先保召回覆盖，再提精度，最后控成本和时延。范式上按场景选 ReAct、Plan-Execute、Router 或 Self-RAG。”

5 分钟版：“我会补充范式取舍和失败治理：路由错误怎么 fallback、重排预算怎么控、证据不足何时拒答，以及如何通过离线回归 + 线上 A/B + 失败回放持续优化。”
